Portable PsyAgent Quick Start Guide

Get up and running with the portable psychological assessment agent system supporting local Ollama models and cloud-based LLMs.

Introduction

Portable PsyAgent is a portable psychological assessment agent system that supports both local Ollama models and various cloud-based LLMs. It provides multi-dimensional personality assessment (Big Five), supports multiple evaluators, and features batch analysis capabilities for large datasets.

Key Features: Multi-dimensional personality assessment, Big Five model, Ollama integration, batch analysis, and detailed reporting.

Prerequisites

1

Install Ollama

Download and install Ollama from https://ollama.ai/download

# Start Ollama service
ollama serve
2

Pull Required Models

Install the recommended models:

ollama pull mistral:instruct
ollama pull phi3:mini
ollama pull qwen3:4b
3

Install Python Dependencies

Install required Python packages:

pip install requests openai anthropic dashscope python-dotenv

Basic Setup

1

Clone the Repository

git clone https://github.com/your-repo/portable_psyagent.git
cd portable_psyagent
2

Configure Environment

Copy .env.example to .env and fill in your API keys:

cp .env.example .env
# Edit .env file to add your API keys
3

Verify Ollama Connection

# Check available models
ollama list

Running Assessments

Single File Analysis

Analyze a single assessment file:

python shared_analysis/analyze_results.py path/to/your/data.json

Using Specific Evaluators

Run with specific evaluators:

python shared_analysis/analyze_results.py data.json --evaluators phi3_mini qwen3_4b

Batch Analysis

Run batch analysis on multiple files:

# View file statistics
python ultimate_batch_analysis.py --stats

# Run quick test (5 files)
python ultimate_batch_analysis.py --quick

# Analyze specific model series
python ultimate_batch_analysis.py --filter deepseek --sample 10

Understanding the Workflow

Analysis Process

The system follows a comprehensive workflow:

  1. Format Conversion: Convert raw assessment data to standard format
  2. Report Simplification: Create simplified versions to avoid context limits
  3. Segmented Analysis: Break data into segments for detailed assessment
  4. Multi-Evaluator Processing: Use multiple evaluators for reliability
  5. Aggregation: Combine results from multiple evaluators
  6. Reporting: Generate detailed analysis reports

Batch Analysis Workflow

For large datasets, the system processes data through:

  • Automatic Format Conversion: Handles multiple input formats
  • Intelligent Batching: Processes files in optimal batch sizes
  • Progress Tracking: Monitors analysis progress and estimated time
  • Error Recovery: Continues processing interrupted analyses

Output and Results

Analysis Output

The system generates comprehensive results:

  • JSON Results: Structured analysis results in JSON format
  • Markdown Reports: Human-readable analysis reports
  • HTML Reports: Formatted reports for easy viewing
  • Evaluation Summaries: Concise summaries of key findings
  • Debug Logs: Detailed logs for troubleshooting

Batch Analysis Output

batch_analysis_results/
├── batch_analysis_summary.json # JSON summary of results
├── batch_analysis_summary.md # Markdown summary
└── [batch_name]/ # Individual batch results
├── analysis_results.json # Raw analysis results
├── analysis_report.md # Human-readable report
└── analysis_report.html # HTML formatted report
